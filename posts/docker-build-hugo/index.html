Docker构建hugo镜像

June 20, 2022
Table of Contents
  • Dockerfile
  • 构建
  • 运行容器

Dockerfile#

创建 Dockerfile ：

FROM ubuntu:latest

LABEL author="king"

#COPY sources.list /etc/apt

EXPOSE 1313

RUN set -ex && \
        sed -i s@/archive.ubuntu.com/@/mirrors.aliyun.com/@g /etc/apt/sources.list && \
        apt clean && \
        apt update && \
        apt -y install hugo

WORKDIR /src/hugo

CMD bash

构建#

$ docker build -t myhugo:latest .

运行容器#

$ docker run -it --rm \
	-p 1313:1313 \
	-v $PWD:/src/hugo \
	--user `id -u`:`id -u`  \
	myhugo \
	hugo server -D --bind 0.0.0.0

  • 默认root用户运行生成的文件权限为root，这里用 --user 参数指定用户 uid:gid 。

  • 容器内应用必须绑定 0.0.0.0，如果是 127.0.0.1 （localhost），将无法访问端口。

  • 0.0.0.0 ，最特殊的一个ip地址，代表的是本机所有ip地址，不管你有多少个网口，多少个ip，如果监听本机的 0.0.0.0 上的端口，就等于监听机器上的所有ip端口。

  • 127.0.0.1 是一个环回地址。并不表示“本机”。0.0.0.0 才是真正表示“本网络中的本机”。
